# Boat 000 — Design Spec

**Date:** 2026-04-21
**Status:** Design — approved, awaiting implementation plan
**Author:** Lachlan + Claude (brainstorming session)
**Sub-project:** 2 of N in the Boat tower family. Sub-project 1 (water surface system) is shipped (commit `90b0020`).

---

## 1. Goal

Add **Boat 000** — the entry-level water-placed tower — to Gloomfell. The base tower is a low gothic galleon that broadsides the closest enemy with twin cannons. This sub-project covers the base tower only (`boat000`); upgrade paths 100 / 010 / 001 (artillery, support, income) are explicitly **out of scope** and become future sub-projects.

Deliverables:
- Blender model + texture/palette → FBX export
- Hybrid animation: shape-key cloth in Blender + C# for everything else (idle, placement, shoot)
- Two SFX cues (placement + fire), generated via ElevenLabs `text_to_sound_effects`
- Wired into `TowerCosts`, `TowerPlacer`, with new `BoatAttack` and `BoatAnimator` components
- Reuses existing `Velocity.cs` for projectiles, existing `SquashStretch` and `SfxPlayer` for feel/audio

Out of scope:
- Upgrade rows in `TowerCosts._upgrades` for `boat000` (no upgrades yet)
- Path 1/2/3 visual variants (boat100.fbx, boat010.fbx, boat001.fbx)
- Impact SFX or custom hit VFX on cannonballs (matches Tack — keeps the mix clean)

## 2. Aesthetic & assets

### 2.1 Model

**Source:** `Assets/Blender/blender_boat000.py` (new)
**Export:** `Assets/Models/boat000.fbx`
**Palette:** Dark gothic galleon

Hierarchy (root: `Boat`):
- `Hull` — clinker-style planked body, ~2.2 long × 0.9 wide × 1.4 tall above waterline; slight upward sheer at bow + stern; bevels + subsurf so silhouette reads as rounded carved wood.
- `DeckRail` — short bevelled railing on top edge of hull
- `MastFore` — fore mast, simple cylinder + bevels
- `MastMain` — main mast, slightly taller than fore
- `SailFore` — billowed quad-mesh sail on `MastFore`, subdivided enough to support the `Ripple` shape key (~8×8 grid)
- `SailMain` — billowed quad-mesh sail on `MastMain`, larger, same subdivision
- `Flag` — small subdivided plane (~6×4 grid) on top of `MastMain`
- `Turret` (empty) — parents `Hull`, `DeckRail`, `MastFore`, `MastMain`, `SailFore`, `SailMain`, `Flag`, `CannonPort_L`, `CannonPort_R`. The whole boat-above-waterline is under this empty so a single yaw rotates everything as one piece.
- `CannonPort_L` — short barrel cylinder on the **port** side amidships, fixed angle pointing along `-Turret.right`
- `CannonPort_R` — short barrel cylinder on the **starboard** side amidships, fixed angle pointing along `+Turret.right`

**Cannon mounting (critical, easy to get wrong):** Both cannons are **rigidly fixed** to the hull at fixed angles. They do **not** rotate independently. To aim, the entire `Turret` (containing hull + masts + cannons) yaws as one piece so the port cannon lines up with the target. Both cannons then fire simultaneously along their fixed barrel axes — the starboard shot goes the opposite direction regardless of whether anything is there.

### 2.2 Palette (baked into Blender materials)

| Part | RGB | Notes |
|------|-----|-------|
| Hull / rails / masts | `(0.10, 0.07, 0.06)` | Near-black stained wood, smoothness ~0.2, metallic 0 |
| Sails / flag | `(0.42, 0.06, 0.08)` | Deep crimson, smoothness ~0.2, metallic 0 |
| Cannons + iron fittings | `(0.18, 0.18, 0.20)` | Tarnished iron, smoothness ~0.4, metallic ~0.3 |

Subtle baked highlight rim along plank edges via vertex paint, so the silhouette pops against the dark blue lake.

### 2.3 Blender → FBX export options

Mirror `blender_lake.py` (these are critical):
```python
bpy.ops.export_scene.fbx(
    filepath=FBX_PATH,
    use_selection=False,
    object_types={'MESH', 'ARMATURE', 'EMPTY'},
    bake_anim=False,
    add_leaf_bones=False,
    axis_forward='-Z',
    axis_up='Y',
    global_scale=1.0,
    bake_space_transform=True,
    apply_scale_options='FBX_SCALE_ALL',
)
```

`bake_space_transform=True` + `apply_scale_options='FBX_SCALE_ALL'` are required for correct orientation + 1:1 scale on import (lesson learned from the lake task — without these, the boat would import at 1/100 scale and rotated wrong).

`object_types` must include `ARMATURE` because sails + flag are exported as `SkinnedMeshRenderer` to enable blendshapes (see Section 4.1).

### 2.4 Import settings

`Assets/Models/boat000.fbx.meta`:
- `isReadable: 0` — blendshapes don't require readable verts; the boat hull doesn't need runtime vertex access either.
- Materials: extract on import, store in `Assets/Materials/boat000/` so they can be tweaked in Unity if needed.
- Mesh compression: default.

### 3.2 BoatAttack.cs (new component, implements ITowerAttack)

```csharp
public class BoatAttack : MonoBehaviour, ITowerAttack
{
    public float range = 5.5f;
    public int damage = 2;
    public int pierce = 1;
    public float cooldown = 1.3f;
    public float projectileSpeed = 18f;
    public float aimSpeed = 90f;        // degrees per second
    public float aimTolerance = 15f;    // degrees
    public GameObject projectilePrefab;

    public float Range => range;

    private Transform _turret;
    private Transform _cannonL;
    private Transform _cannonR;
    private float _cdTimer;
    private SquashStretch _squash;
    private BoatAnimator _animator;
    // ...
}
```

**Lifecycle:**
- `Start()`: cache `_turret = transform.Find("Turret")`, `_cannonL = ...Find("Turret/CannonPort_L")`, `_cannonR = ...Find("Turret/CannonPort_R")`, `_squash = GetComponent<SquashStretch>()` (added by `TowerPlacer._placementSetup` like every other tower), `_animator = GetComponent<BoatAnimator>()`. Initialize `_cdTimer = 0`.
- `Update()`:
  1. Find closest enemy in `Spawn.UnitsParent` within `range`. If none, idle (no rotation, no fire).
  2. Compute the XZ direction inline:
     ```csharp
     Vector3 d = target.position - _turret.position;
     d.y = 0f;
     Vector3 toTarget = d.normalized;
     ```
  3. Desired port direction is `-_turret.right`. Rotate `_turret` toward a rotation where `-right` aligns with `toTarget`, capped at `aimSpeed * Time.deltaTime` via `Quaternion.RotateTowards`.
  4. If `Vector3.Angle(-_turret.right, toTarget) < aimTolerance` and `_cdTimer <= 0`, call `Fire()`.
  5. Decrement `_cdTimer` by `Time.deltaTime`.

**Fire():**
- Spawn projectile at `_cannonL.position`, direction `-_turret.right`.
- Spawn projectile at `_cannonR.position`, direction `+_turret.right`.
- Each projectile is `Instantiate(projectilePrefab)` then `Velocity` component configured: `direction`, `speed = projectileSpeed`, `damage`, `pierce`, `lifetime = range + 1`.
- `_squash.Play()` (matches existing `TackAttack` API).
- `_animator.PlayShoot()`.
- `SfxPlayer.PlayOneShot("SFX/boat_fire", transform.position, volume: 0.85f, pitchJitter: 0.05f)` — see Section 5.4 for the SfxPlayer extension this requires.
- `_cdTimer = cooldown`.

**SquashStretch wiring (in `Start`):**
Following the existing `TackAttack` pattern (`TackAttack.cs:23-39`), `BoatAttack.Start()` adds `SquashStretch` itself, walks children, and skips parts that should not be squashed. Skip prefixes for the boat: `Hull`, `DeckRail`, `MastFore`, `MastMain`, `Turret` are squashed; `CannonPort_L`, `CannonPort_R`, `SailFore`, `SailMain`, `Flag` are skipped (their own animations would clash with the squash). **No edits to `SquashStretch.cs` are required** — the exclusion list lives per-tower in the attack component's `Start`, matching the existing pattern.

## 4. Animation (hybrid)

### 4.1 Authored in Blender

**Sail ripple** (on `SailFore`, `SailMain`):
- Add a Wave modifier in `blender_boat000.py` with parameters tuned to give a small visible cloth ripple along the sail's local Y axis (amplitude ~0.04, wavelength ~0.5).
- Bake the wave at frame 30 of one full cycle into a single shape key named `Ripple`.
- Remove the modifier afterwards. Final mesh has only the shape key.

**Flag wave** (on `Flag`):
- Same recipe with larger amplitude (~0.10).
- Vertex group `MastSide` weights = 1 on the corner attached to the mast, falling to 0 across the flag. Wave modifier respects this so the corner stays anchored.
- Bake into single `Ripple` shape key.

**Skinning trick:** Both meshes need `SkinnedMeshRenderer` in Unity to support blendshapes. Add a single dummy bone (single-vertex armature) parented to the mesh, weight all vertices to it, and include `'ARMATURE'` in `object_types` on FBX export. Unity will import as `SkinnedMeshRenderer` automatically.

### 4.2 BoatAnimator.cs (new component)

Drives idle / placement / shoot. Single `Update()` switches on `_state` (Idle / Placing). Shoot is one-shot tweens layered on top.

**Cached refs (in `Start`):** `_hull`, `_mastFore`, `_mastMain`, `_sailFore` (SkinnedMeshRenderer), `_sailMain` (SkinnedMeshRenderer), `_flag` (SkinnedMeshRenderer), `_cannonL`, `_cannonR`.

**Idle (continuous, runs every frame in Idle state):**
```csharp
float t = Time.time;
_hull.localPosition = _hullRest + Vector3.up * Mathf.Sin(t * 1.4f) * 0.05f;
_hull.localRotation = _hullRestRot * Quaternion.Euler(0, 0, Mathf.Sin(t * 1.0f) * 3f);
_mastFore.localRotation = _mastForeRest * Quaternion.Euler(0, 0, Mathf.Sin(t * 1.2f) * 1.5f);
_mastMain.localRotation = _mastMainRest * Quaternion.Euler(0, 0, Mathf.Sin(t * 1.2f + 1.7f) * 1.5f);
_sailFore.SetBlendShapeWeight(0, (Mathf.Sin(t * 2.0f) * 0.5f + 0.5f) * 100f);
_sailMain.SetBlendShapeWeight(0, (Mathf.Sin(t * 2.0f + 0.5f) * 0.5f + 0.5f) * 100f);
_flag.SetBlendShapeWeight(0, (Mathf.Sin(t * 3.0f) * 0.5f + 0.5f) * 100f);
```

**Placement (one-shot, called by the setup action — see Section 3.4 — via `PlayPlacement()`):**
- Duration 0.6s; sets `_state = Placing`.
- `transform.position.y` starts `+1.5` above the placement point, eases to placement Y via `SmoothStep`.
- `transform.localScale` starts `(1.1, 1.1, 1.1)`, lerps to `(1, 1, 1)`.
- At impact frame (when easing hits placement Y): `Instantiate(splashRingPrefab, transform.position, Quaternion.identity)` (the boat's `transform.position` is the waterline since placement is on the lake mesh) and `SfxPlayer.PlayOneShot("SFX/boat_place", transform.position, volume: 0.7f)`.
- After 0.6s: `_state = Idle`.

**Shoot (one-shot, called by `BoatAttack.Fire()` via `PlayShoot()`):**
- Cannon recoil: tween `_cannonL.localPosition` back along its barrel axis (local `+Z` if barrel points `-Z`) by 0.15 over 0.05s, then ease forward over 0.3s. Same for `_cannonR`.
- Smoke puff: `Instantiate(muzzleSmokePrefab, _cannonL.position, _cannonL.rotation)` and same at `_cannonR`. Particle system one-shot, 8 particles, 0.5s lifetime, drifts up.
- Mast sway impulse: add a transient `+2°` Z-rotation to `MastFore` and `MastMain`, decaying over 0.4s (track an `_impulseDecay` float per mast, decremented in `Update`, added to the idle sway).
- Deck splash ripple: small `Instantiate(splashRingPrefab, transform.position, Quaternion.identity)` with scale 0.5× and lifetime 0.2s.

**Prefab references:** `splashRingPrefab` and `muzzleSmokePrefab` are `[SerializeField] GameObject` fields on `BoatAnimator`, assigned on the **`Boat000.prefab`** (not via `_placementSetup`). This means `BoatAnimator` must already be a component on the prefab with refs wired in the editor — see Section 3.4 for the prefab vs setup-action split.

### 4.3 SquashStretch exclusion (per-tower, in BoatAttack.Start)

See Section 3.2 "SquashStretch wiring." The exclusion list is implemented per-tower by `BoatAttack.Start()` walking children and skipping by name prefix — matching the existing pattern in `TackAttack.cs:23-39`. **`SquashStretch.cs` itself is not modified.** The pulse on the rest of the boat (hull, masts, deck rail, turret) reads as the whole hull bunching up under recoil.

### 4.4 New VFX prefabs

`Assets/Prefabs/VFX/SplashRing.prefab`:
- Small horizontal torus mesh (Blender-authored or Unity primitive Torus scaled flat)
- URP/Lit unlit-ish material, white, alpha 0.6 → 0
- Script: `SplashRing.cs` (new) — over its lifetime, scale.x and scale.z lerp from 0 to 1, alpha fades, `Destroy(gameObject)` at end. Lifetime configurable (placement uses 0.4s, shoot uses 0.2s).

`Assets/Prefabs/VFX/MuzzleSmoke.prefab`:
- ParticleSystem, single emission burst of 8 particles, 0.5s lifetime, grey color (`0.5, 0.5, 0.5, 0.5` → transparent), drift `+Y` at 0.5 units/s, slight outward velocity along `-Z` (cannon barrel direction).
- `Destroy(gameObject)` after 1.0s (giving particles time to clear before the GO is removed).

## 5. Audio (SFX)

Two cues, both generated via ElevenLabs `text_to_sound_effects`, saved as `.mp3` into `Assets/Resources/SFX/`. Played by the existing `SfxPlayer`.

### 5.1 boat_place.mp3
- **Prompt:** *"Wooden hull splashing down into calm water, brief water surge, single thump of impact, short and punchy. ~0.6 seconds."*
- **Duration:** ~0.6s
- **Trigger:** `BoatAnimator.PlayPlacement()` impact frame
- **Volume:** 0.7
- **Pitch jitter:** none

### 5.2 boat_fire.mp3
- **Prompt:** *"Twin black-powder cannon broadside fired in unison, deep low-end boom, brief crack, short tail of smoke hiss, no echo. ~0.5 seconds."*
- **Duration:** ~0.5s
- **Trigger:** `BoatAttack.Fire()` (every volley)
- **Volume:** 0.85
- **Pitch jitter:** ±5% (single sample, randomized per play to avoid monotony)

### 5.3 No impact SFX
Cannonballs make no sound on hit — matches Tack and keeps the audio mix from getting noisy when many towers fire.

### 5.4 SfxPlayer extension (small modification)

Current `SfxPlayer.PlayOneShot(resourcePath, worldPos, volume)` does not support pitch jitter. Add an optional `pitchJitter = 0f` parameter that, when non-zero, sets `src.pitch = 1f + Random.Range(-pitchJitter, pitchJitter)` before `src.Play()`. Default of 0 keeps every existing call unchanged.

```csharp
public static void PlayOneShot(string resourcePath, Vector3 worldPos, float volume = 1f, float pitchJitter = 0f)
{
    // ... existing body ...
    src.volume = volume;
    if (pitchJitter > 0f) src.pitch = 1f + Random.Range(-pitchJitter, pitchJitter);
    // ... rest unchanged ...
}
```

This is the only modification to `SfxPlayer.cs`.
